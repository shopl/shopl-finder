## 주요 기능

<!-- 이미지 필요: 전체 기능을 보여주는 메인 스크린샷 -->

SHOPL 대시보드는 JSP와 React가 혼합된 하이브리드 구조로 되어 있어 **"이 화면이 어떤 기술로 만들어졌는지"** 파악하기 어려울 수 있습니다.<br/>
SHOPL 페이지 분석기는 이런 복잡한 구조를 **색깔로 구분**해서 보여주어, 비개발 직군도 쉽게 이해할 수 있게 도와주는 **시각화 도구**입니다.

### 기술 스택 하이라이팅

**"이 화면이 JSP인지 React인지 궁금할 때"** 사용하는 기능입니다. 각 영역을 색상으로 구분하여 한눈에 파악할 수 있습니다.

| **유형** | **설명** | **분석 결과** |
|------|---------|-------------|
| **JSP Page** | 전체가 JSP로 개발된 페이지/모달 | 🟧 주황색으로 표시 |
| **React Page** | 전체가 React로 개발된 페이지/모달| 🟦 파란색으로 표시 |
| **React Modal** | JSP 페이지 내에 있는 React 모달 | 모달 영역만 🟦 파란색 |
| **React Section** | JSP 페이지 내 부분적 React 적용 영역 | 해당 섹션만 🟦 파란색 |

<!-- 이미지 필요: 기술 스택 하이라이팅 결과 스크린샷 -->

**언제 사용하면 좋을까요?**
- **기획자**: 기능 개선 요청 시 어떤 기술로 개발되었는지 확인. 개발 우선순위 결정을 위한 기술 현황 파악
- **디자이너**: 디자인 수정 요청 범위 파악
- **QA**: 이슈 담당자 파악
- **비즈니스팀**: 고객 문의 대응 시 기술적 배경 이해

**주요 특징:**
- 페이지 내 모든 React 컴포넌트 자동 탐지
- iframe 내부 요소까지 깊이 분석
- 하이브리드 구조의 복잡성을 직관적으로 시각화

### 디자인시스템(Shoplflow) 하이라이팅

**"이 화면에서 디자인시스템이 얼마나 잘 적용되었는지"** 확인하는 기능입니다. 통일된 디자인 적용 현황을 한눈에 파악할 수 있습니다.

<!-- 이미지 필요: Shoplflow 하이라이팅 결과 스크린샷 -->

**언제 사용하면 좋을까요?**
- **기획자**: 사용자 경험 통일성 현황 파악
- **디자이너**: 디자인시스템 적용률 확인 및 일관성 검토


**주요 특징:**
- shoplflow 컴포넌트 자동 탐지
- 디자인팀과 개발팀이 합의한 표준 컴포넌트만 표시

**🔍 탐지 기준:**
- ✅ **포함**: 디자인팀과 합의된 표준 컴포넌트
- ❌ **제외**: 개발팀 전용 기술 컴포넌트 (`Stack`, `Popper`, `StackContainer`, `shopl`, `BackDrop`)

> ⚠️ **참고**: SHOPL Finder는 개발-비개발 직군의 협업 도구이므로, 실제 사용자가 보는 디자인 요소만 찾아서 보여줍니다.

### 초기화

**분석이 끝난 후 페이지를 깔끔하게 정리**하는 기능입니다. 모든 하이라이팅과 분석 결과를 제거하여 원래 상태로 되돌립니다.

**초기화 범위:**
- 모든 색상 하이라이팅 제거
- 호버 시 나타나는 라벨 제거
- iframe 내부 요소까지 완전 초기화

## 요약 팝업

분석 결과를 **간단한 숫자와 메시지**로 요약해서 보여주는 팝업창입니다. 복잡한 기술 정보를 누구나 이해할 수 있게 정리해드립니다.

<!-- 이미지 필요: 팝업 UI 전체 스크린샷 -->


**3개 주요 버튼:**
- **⚙️ 기술 스택 찾기**: JSP/React 영역을 색상으로 구분하여 표시
- **🛍️ Shoplflow 찾기**: 디자인시스템 컴포넌트를 하이라이팅하여 표시
- **🔄 초기화하기**: 모든 하이라이팅을 제거하고 원래 상태로 복원

## 설치 및 사용 방법

### 📥 설치하기
1. [크롬 웹스토어 링크](https://chromewebstore.google.com/detail/shopl-%ED%8E%98%EC%9D%B4%EC%A7%80-%EB%B6%84%EC%84%9D%EA%B8%B0/knimkfpaanfddllbkhbdopcimdangdaa?hl=ko&authuser=0)를 클릭해서 이동합니다.
2. **`Chrome에서 사용`** 버튼을 클릭합니다.
3. 팝업이 뜨면 **`확장 프로그램 추가`** 버튼을 클릭해서 설치를 완료합니다.

### 🚀 기본 사용법
1. **SHOPL 대시보드 페이지**에 접속합니다. (개발/QA/운영 서버 모두 지원)
2. 브라우저 우측 상단의 **확장 프로그램 아이콘** 중 **SHOPL 페이지 분석기**를 클릭합니다.
3. 원하는 기능을 선택해서 분석을 시작합니다.
4. 분석 완료 후 **🔄 초기화하기** 버튼으로 깔끔하게 정리합니다.

### 📋 기능별 상세 가이드

#### ⚙️ 기술 스택 찾기 사용법
**목적**: 페이지의 JSP/React 구조 파악

1. **⚙️ 기술 스택 찾기** 버튼 클릭
2. 잠시 기다리면 페이지가 색상으로 구분됩니다
3. **결과 확인**:
   - 🟦 파란색 = React 영역
   - 🟧 주황색 = JSP 영역
4. **활용 방법**:
   - 마우스를 올리면 더 자세한 정보 확인 가능
   - 스크린샷으로 저장해서 팀원들과 공유
   - 기능 개선 요청 시 참고 자료로 활용

**💡 이런 경우에 유용해요:**
- 기능 추가/수정 요청 전 기술 현황 파악
- 개발 난이도 예상 및 일정 산정 참고
- 팀 간 커뮤니케이션 시 구체적 설명

#### 🛍️ Shoplflow 찾기 사용법
**목적**: 디자인시스템 적용 현황 파악

1. **🛍️ Shoplflow 찾기** 버튼 클릭
2. 디자인시스템이 적용된 컴포넌트가 하이라이팅됩니다
3. **결과 해석**:
   - 하이라이팅된 요소 = 표준 디자인 적용됨
   - 하이라이팅 안된 요소 = 커스텀 또는 레거시 디자인
4. **활용 방법**:
   - 디자인 일관성 현황 체크
   - 리뉴얼 우선순위 결정 참고
   - 브랜드 가이드라인 준수 현황 확인

**💡 이런 경우에 유용해요:**
- 디자인 QA 및 일관성 검토
- 리브랜딩 또는 디자인 개선 계획 수립
- 사용자 경험 표준화 현황 파악

#### 🔄 초기화하기 사용법
**목적**: 분석 후 원래 화면으로 복원

1. **🔄 초기화하기** 버튼 클릭
2. 모든 하이라이팅과 라벨이 제거됩니다
3. **언제 사용하나요?**:
   - 분석 완료 후 깨끗한 화면 복원
   - 다른 기능 사용 전 초기화
   - 실제 사용자 화면 확인 시

### 🔧 문제상황별 대응법

**결과가 나오지 않을 때:**
- 페이지가 완전히 로드된 후 다시 시도해보세요
- iframe이 많은 페이지는 로딩에 시간이 걸릴 수 있습니다
- 브라우저를 새로고침한 후 다시 시도해보세요

**하이라이팅이 이상할 때:**
- 🔄 초기화하기로 리셋 후 다시 시도해보세요
- 다른 확장 프로그램과 충돌할 수 있으니 비활성화 후 테스트해보세요

**팝업이 열리지 않을 때:**
- 확장 프로그램이 SHOPL 도메인에서만 작동하는지 확인해보세요
- 브라우저 확장 프로그램 설정에서 활성화 상태를 확인해보세요

## 주의사항
- 🏢 **회사 내부 전용**: SHOPL 대시보드 페이지에서만 작동합니다
- 🔄 **초기화 권장**: 분석 후에는 **🔄 초기화하기**로 페이지를 원래 상태로 복원해주세요
- 📱 **성능 고려**: 복잡한 페이지에서는 분석에 시간이 걸릴 수 있습니다
- 🌐 **지원 환경**: Chrome 브라우저에서만 사용 가능합니다

## 문제해결 및 피드백
다음과 같은 경우 **사내 메신저**를 통해 **프론트엔드팀**으로 연락해주세요:

**🚨 긴급 문제:**
- 익스텐션이 전혀 작동하지 않는 경우
- 페이지가 깨지거나 오류가 발생하는 경우

**🔍 분석 관련:**
- 특정 페이지에서 영역 구분이 잘못 표시되는 경우
- 예상과 다른 결과가 나오는 경우

**💡 개선 제안:**
- 새로운 기능 아이디어가 있는 경우
- 사용성 개선 제안이 있는 경우
- 추가로 분석하고 싶은 요소가 있는 경우

**📞 연락 시 포함해주세요:**
- 사용 중인 브라우저 버전
- 문제가 발생한 페이지 URL
- 스크린샷 (가능한 경우)
